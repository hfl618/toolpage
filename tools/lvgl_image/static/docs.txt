# LVGL 图像转换引擎 - 使用指南

## 1. 核心转换参数
- **LVGL 版本**: 支持 v9 (最新) 和 v8 (兼容)。v9 使用全新的色彩格式宏，v8 会自动转换变量命名。
- **色彩格式**: 
  - `RGB565A8`: LVGL 特色格式，16位色+8位独立 Alpha 映射，平衡性能与显示效果。
  - `I1/I2/I4/I8`: 索引色模式，使用调色板，极大地减少内存占用。
  - `RAW`: 不进行转码，直接将原图转为 C 数组，需配合图片解码器使用。
- **内存对齐**:
  - `4 Bytes`: 标准对齐，适用于大多数 ARM 芯片。
  - `32 Bytes`: 适配带 Cache Line 的高性能芯片（如 i.MX RT10xx），防止缓存污染。
  - `64 Bytes`: 极致性能对齐，满足某些高速图形加速器的爆发传输要求。

## 2. 增强功能
- **Dithering (抖动)**: 在 16 位色（RGB565）下建议开启，能有效消除渐变色的断层现象。
- **Premultiply Alpha (预乘 Alpha)**: 提前将 Alpha 合并到 RGB 中，减少 MCU 在渲染时的实时浮点运算量。
- **NEMA 加速优化**: 专门针对支持 NEMA GFX 加速器的芯片（如 Renesas、NXP 部分系列）进行的字节序和调色板优化。

## 3. 注意事项
- **输出命名**: 建议使用小写字母和下划线（如 `ui_img_logo`）。若留空，系统将使用图片原始文件名。
- **输出尺寸**: 留空则保持原图比例。若只填宽或高，系统将自动等比例缩放。
- **LVGL v8 提示**: 若目标工程为 v8，请尽量避免使用 v9 独有的复杂色彩格式。
